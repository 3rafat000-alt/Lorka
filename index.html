طبقة النزاهة والذكاء · الدستور v6

البنية بُنيت. والطبقة التي تجعلها تُصدَّق صارت دستوراً.The structure was rebuilt. The layer that makes it trustworthy is now constitutional law.

v5 أضاف طبقة النزاهة فوق 30 وكيلاً؛ v6 أعاد بناء الشركة كاملةً — 105 وكلاء في 15 غرفة — وثبّت الطبقة نفسها قانوناً ملزماً في company/constitution/: أيّ وكيل يقدر أن يهلوس بثقة ويعتبر «تمّ» دليلاً — لولا هذه المواد.

C1

التأريض — استشهد أو اصمتGrounding

خمس قواعد ملزمة لكل وكيل في كل دورة: استشهد بـ ملف:سطر أو ملف الدماغ أو كوميت — وإلا اكتب «غير مؤكَّد» وتوقّف؛ الامتناع مكافأة لا فشل؛ لا تدّعِ تنفيذاً لم تشاهده؛ افصل «تحقّقت» عن «استنتجت»؛ أظهِر التعارض بدل حسمه بصمت.

C2

الانعكاس — منظومة تحلم بتاريخهاReflection · "dreaming"

محرّك مجدول (لا لكل دورة) يقرأ تاريخ HANDOFFS ويقطّره إلى دروس إجرائية دائمة في LESSONS.md عبر /sofi-reflect. كل درس سطر: الموقف · ما فشل · القاعدة. الاحتفاظ افتراضي — لا يُمحى تاريخ خام.

C4

التحقّق — النتيجة لا الإفادة الذاتيةVerification

تذكرة «منجزة» بلا كتلة دليل (أمر + مخرجات + كود خروج) يرفضها gate-check آلياً. وقبل تقدّم أي بوابة، وكيل مستقل بسياق جديد يراجع ضد معايير التذكرة الأصلية فقط — بأدوار ثابتة، و«غير معروف» حكم صالح دائماً.

C3

عقل قابل للاستعلامStructured queryable brain

حقول frontmatter خفيفة (status·gate·type·mem·date) تحوّل التذاكر والدروس إلى سجلات قابلة للفلترة حسب نوع الذاكرة — دلالي · حَدَثي · إجرائي — بلا قراءة الملف كاملاً كل مرة.

C5

استقلالية محسوبة الميزانيةBudgeted autonomy + effort-scaling

جدول يربط فئة المهمة (تافهة · دور واحد · عابرة للغُرف · مسح · تحكيم) بعرض التفريع وسقف النداءات — يمنع الإفراط والتفريط. كل تفويض يحمل تلميح جهد وشرط توقف آمن.

C6

أمر العمل RCCFRCCF Work Order

كل استدعاء بلوك من أربعة: الدور · السياق · الأمر · الصيغة (company/constitution/01-work-order.md) — مع فرع أسئلة توضيحية قبل بريف ناقص، وتجميد البريف بعد اكتماله، وحقول صريحة للحدود وتلميح الجهد وبند التأريض.

طبقة الميكنة · v6.1

الدوكترين لم يعد نصّاً يُوعَظ به — صار كوداً يَحجُب ويَفحَص ويُحاسِب.The doctrine is no longer prose to preach — it is code that blocks, lints, and budgets.

بحثٌ بثلاثة عشر وكيلاً في مستودعات claude على GitHub خلص إلى أن مشاريع مستقلّة وصلت لنفس عقيدة سوفي — فالدلتا ليست عقيدةً جديدة بل ميكنتها. تسع وحدات Python حتمية (النموذج خارج مسار الكتابة) تُحوّل القوانين إلى آلات: company/os/sofi_tools/.

scheduler

المجدول الحتميsofi plan → DAG → sofi run

نداءٌ ذكيٌّ واحد يُفكّك الخطة إلى DAG مجمّد في الدماغ؛ ثم الإرسال والفحوص والدمج تجري بـ Python خالص — صفر توكن على التنسيق، أكبر رافعة على مستوى 105 وكلاء.

transitions

بوّابات fail-closedgate state machine

آلة حالات على البوابات 0–8: تقدُّم / إعادة عمل / حلقة رجوع — وأيّ قفزة +2 = ILLEGAL_SKIP. خطّاف التوقّف يطلب دليلاً عند حدّ البوابة، fail-open ومحصور بالمشروع بسقف ضدّ الجمود.

budget

ميزانيات عند الاستدعاءspawner-level caps

سقوف التزامن والجولات والتوكن والمهلة تُفرَض حيث يُولَد العمل، لا بانضباط الوكيل. نبضٌ لكل سرب، وقاطع دائرة بعد ٣ محاولات يكتب crash-dump مهيكلاً.

memdb

خط دماغ SQLite FTS5the harness remembers

PostToolUse يلتقط، SessionStart يحقن ملخّصاً ≤1k توكن، والفهرسة على العناوين ببصمة SHA-256 لكشف الانحراف؛ البحث يُرجع فهرساً لا الملف كاملاً (~12k ← ~400 توكن). النموذج خارج مسار الكتابة.

lessons_cache

الدروس كذاكرة مُدارةmanaged cache, not a log

درجات ثقة، سقف حقن (≤6 فوق 0.7)، ترقية بعتبة التكرار (5+ عبر مشروعين → عالمي)، تحلّل 5%/أسبوع، ولقاحٌ يُحقن الدرس المطابق في أمر العمل الوارد.

acceptance

موجّه قبول بصفر توكنzero-cost accept router

يُجرّب الطبقة الرخيصة، يُقيّم المخرَج آلياً (رفض/بتر/فشل تحليل مقابل 0.80)، ويصعّد بدليل — «التصعيد بدليل فقط» صار ميكانيكياً؛ وقواعد تجاوز صلبة (أمن/مال/PII) تسبق المصنّف.

resume

استئناف بـ breadcrumbFRESH / DEGRADED

لكل تذكرة breadcrumb.json يحمل head_sha؛ عند الاستئناف: FRESH إن طابق HEAD (استئناف حتمي) أو DEGRADED (إعادة بناء + تأكيد صريح) — لا استئناف صامت على حالة قديمة.

agentlint

فحص الإعدادات في CIconfig lint

يتحقّق من كل .claude/agents/*.md: الترويسة، تطابق 105↔105، اتّساق السجل، وتثبيت SHA-256 ضدّ الانحراف — عبر sofi lint و GitHub Actions.

telemetry

قياس أسطول عن بُعدfleet telemetry

كل خطّاف يبثّ حدثاً إلى سجل JSONL + خادم أحداث بمكتبة قياسية؛ الداشبورد يعرضه في تبويب «الأسطول والخطة» حيّاً. عند 105 وكلاء، الخطاطيف تبثّ كما تَحرُس.

العقيدة والتوجيه الاقتصادي

العقيدة صخرة — كل قاعدة تعود إليها.The doctrine is bedrock — every rule traces back to it.

  • I

    التصميم هو الحقيقةDesign is Truth

    لا سطر كود بلا خطوة في خريطة الرحلة. المصمّمون يقودون، والمهندسون يترجمون.

  • II

    التدفّق الهرميHierarchical Flow

    العمل ينزل بالترتيب، لا تُقفز بوّابة، والعمل الناقص يُرفَض لأعلى — لا ارتجال.

  • III

    العزل الجذريRadical Isolation

    كل مشروع في فضائه الخاص — صفر تسرّب. الكود المشترك في shared-packages/ فقط.

  • IV

    اقتصاد التوكنToken Economy

    أرخص نموذج وأقل جهد وأكثف مخرَج يجتاز المعيار. الهدر عيب، التصعيد بدليل فقط.

  • V

    التحوّل المستمرContinuous Metamorphosis

    بيانات الإنتاج تغذّي الدورة التالية. المنتج لا يكتمل — يتطوّر.

  • VI

    قابلية التراجعReversibility

    الرخيص التراجع يُفوَّض بسرعة؛ الغالي يستحق التمهّل و ADR وخطة تراجع موثّقة.

  • VII

    حلقة الأوراكل الذاتيةAutonomous Oracle Loop

    نقاط القرار تذهب لمكتب المراجعة الخارجي المعقَّم، لا للإنسان. عمل ← تقرير ← أوراكل ← تنفيذ ← تكرار؛ لا يُقطَع للمستخدم إلا عند فعلٍ هدّام.
