# Reducing Hallucination in Agentic LLM Systems

## State of the art (2025-2026)

The frontier treats hallucination as four distinct, separately-detected failure modes — **factual, grounding, citation, and reasoning** — not one blob. Anthropic's own guidance (Citations API + hallucination-reduction docs) and academic surveys converge on the same doctrine: *give the model the source, force it to point at the source, and let it decline when the source isn't there.*

Key developments:
- **Citations API (Anthropic, 2024-2025)**: instead of prompting for quotes, the API grounds generation at the token level — the model can only assert claims traceable to supplied document spans, and returns structured citation objects. One cited deployment cut source hallucinations from 10% to 0% and raised references-per-response 20%. This is grounding-by-construction, not grounding-by-instruction.
- **Chain-of-Verification (CoVe, Dhuliawala et al. 2023, arXiv:2309.11495)**: generate → draft verification questions against the draft's own claims → answer those questions *independently* (without seeing the original draft, to avoid confirmation bias) → revise. The independence step is the load-bearing part; naive self-critique in the same context window re-hallucinates the same errors.
- **Agentic hallucination taxonomy (arXiv:2509.18970)**: for multi-step agents, hallucination is no longer just "wrong fact" — it's 5 stage-specific failure classes: reasoning (flawed plans), execution (claims a tool succeeded when it didn't / calls nonexistent tools), perception (misreads environment/tool output), memorization (trusts stale/corrupted memory without revalidation), communication (fabricated inter-agent messages in multi-agent systems). Critically, errors **propagate and compound across steps** — an early planning hallucination becomes a "confirmed fact" three tool calls later. This is the single most relevant finding for orchestrator-style systems like SOFI.
- **Abstention is not a free byproduct of capability — it regresses under RL/reasoning tuning.** AbstentionBench (arXiv:2506.09038) shows reasoning-tuned models are *worse* at recognizing unanswerable/underspecified questions than base models — they'd rather generate a confident chain-of-thought than say "insufficient information." This means "make the model smarter" does not fix over-confidence; abstention has to be engineered and rewarded explicitly (see Abstain-R1, arXiv:2604.17073, which RL-trains calibrated refusal + clarification-seeking as its own objective).
- **Self-consistency / best-of-N as a cheap uncertainty proxy**: sample the same prompt N times (or route through independent reasoning paths); divergence across samples is treated as a confidence signal, convergence as corroboration. Anthropic explicitly recommends this ("Best-of-N verification") as a lightweight technique.
- **Multi-agent critique/debate as verification, not just generation**: a generator + independent critic/reflector pair (LangGraph reflection pattern), or adversarial multi-agent debate for fact-checking, outperforms single-pass self-correction because the critic doesn't inherit the generator's blind spots — provided it's a genuinely separate context/pass, not the same agent re-reading its own output in-context.
- **Faithfulness ≠ factuality**: a claim can be internally consistent with the source (faithful) yet wrong (unfactual), or true yet unsupported by given context (unfaithful hallucination). Production RAG pipelines increasingly attach a dedicated faithfulness judge to every generation span and gate the response on that judge before it ships — this is now table stakes, not a nice-to-have.

## Concrete techniques worth adopting

1. **Extract-quotes-before-answering** — for any task over >20k tokens of source material, force a first pass that pulls verbatim quotes, then answer using only those quotes. Grounds the answer in text that actually exists; cheap (no extra API surface needed, pure prompting).
2. **Cite-then-verify-retraction loop** — require every asserted claim to carry a supporting quote/citation; if no quote can be found post-hoc, the claim must be deleted (marked `[]`), not softened. Turns "sounds right" into "provably right or removed."
3. **Explicit permission-to-abstain in the prompt/contract** — state that "I don't have enough information" is a *preferred*, not penalized, output. Directly counters the RL-induced abstention regression AbstentionBench documents — you cannot assume a capable model will volunteer uncertainty; it has to be told this is a rewarded path.
4. **Chain-of-Verification (independent re-check)** — draft → generate verification questions about the draft's own claims → answer them in a *fresh context* that doesn't see the draft → reconcile. The "fresh context" requirement is the key engineering detail most naive implementations skip.
5. **Faithfulness gate as a pipeline stage** — attach a lightweight judge (rule-based or model-based) between "agent produced output" and "output is accepted," specifically checking claim↔source alignment, not just plausibility. Fail-closed: ungrounded claim blocks handoff.
6. **Stage-tagged hallucination taxonomy for multi-step agents** — classify failures by *where* in the pipeline they occurred (plan/reasoning, tool-execution, perception/read of tool output, memory-recall, inter-agent message) rather than lumping all wrongness together. This determines which specialist/verifier should catch it and prevents propagation — catch reasoning hallucinations at the plan gate before they become "facts" downstream.
7. **Execution-claim verification** — for agentic tool use specifically: never trust "I did X" as ground truth; require the tool's actual return value/exit code/diff as the source of truth for whether X happened. This is the single highest-value agentic-specific technique (arXiv:2509.18970's "execution hallucination" is exactly an agent asserting success it never checked).
8. **Best-of-N / self-consistency as a cheap confidence signal** — for high-stakes, low-cost-to-resample decisions, sample N times and treat disagreement as a flag for escalation/abstention rather than silently picking one.
9. **Independent critic pass, not in-context self-review** — route verification through a separate agent/context (different model tier or at minimum a fresh conversation) rather than "now check your own work" in the same transcript, which inherits the same blind spots.

## How it applies to SOFI

**What SOFI already does right (structurally, if not doctrinally):** the tier-isolation + Advisor-gateway pattern is *already* an independent-critic architecture in miniature — cross-tier work forces a second, differently-scoped agent to touch the work before it moves, which is close to the multi-agent-critique pattern above. The 9-gate lifecycle with gate-bars is a real faithfulness-gate mechanism (deliverables must exist and validate before advancing) — this is the "attach a judge, fail-closed" pattern, just not framed as anti-hallucination. The git-backed 4-file brain (STATE/CONTEXT/DECISIONS/HANDOFFS) with `head_sha` checkpointing is a grounding mechanism: agents are supposed to read the actual current state rather than assume/recall it, which directly guards against "memorization hallucination" (arXiv:2509.18970's category 4).

**What's genuinely missing** (confirms the prior DEVELOPMENT-PLAN gap #1 and sharpens it): there is no codified rule that (a) agents must cite `file:line` / brain-file-line for factual claims about the codebase (SOFI's own skills like `sofi-spec-review` already do this informally — "cite file:line, rank by severity" — but it's not a cross-cutting doctrine in `engine/protocols/00-operating-system.md`); (b) agents are explicitly permitted/required to say "insufficient information, escalate" rather than fabricate a plausible-sounding answer under delegation pressure — RCCF's Command block currently has no abstention clause; (c) execution-claim verification is not codified — a dev agent reporting "tests pass" or "migration applied" should be required to paste the actual command output/exit code into the handoff artifact, not just assert it (this maps 1:1 onto arXiv:2509.18970's "execution hallucination," and SOFI's own gate-bars are exactly the enforcement point for this). Concretely: add a **grounding clause to RCCF's Format block** ("cite file:line or brain-file for every factual claim; if ungrounded, say so and stop") and a **gate-bar check** that rejects handoffs asserting "done"/"passing" without pasted command output — this is cheap (a doc + one Python check in `sofi_tools.gates`), directly reuses existing infrastructure, and closes the exact gap the prior analysis flagged.

**What would be over-engineering for SOFI right now:** a dedicated ML-trained abstention model (Abstain-R1-style RL) or conformal-prediction calibration — SOFI has no model-training loop and routes through off-the-shelf Claude tiers, so this is inapplicable. Likewise, a formal multi-agent debate/adversarial-fact-check subsystem is redundant given tier isolation already forces independent review; adding a third verification layer on top of Advisor-gateway + gate-bar would be doctrine-bloat without evidence SOFI's current failure rate justifies it. Best-of-N sampling for every delegation would blow the token-economics ladder (`haiku→sonnet→fable→opus`) SOFI is explicitly optimized against — reserve it (if adopted at all) for Fable-tier arbitration only, where the cost is already justified.
